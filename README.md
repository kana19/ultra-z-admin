# ultra-z-admin

**ウルトラZAIMUくんレオ 運営ポータル**
／ 株式会社ターゲット社 専用管理画面（PWA静的サイト）

レオ・アストラユーザー（`clientId = uz-XXXXXXXX`）を元締め企業ターゲット社が
管理するためのPC専用ポータルです。利用者側PWA（`ultra-z-leo` 系列）とは別系統。

- 公開URL：<https://kana19.github.io/ultra-z-admin/>
- 認証基盤：マスタスプレッドシート `ultra-z-master`（k@tgx.jp 所有）＋ マスタGAS
- ブランド：ウルトラの父
- 基調色：宇宙色 `#0B1842`
- 対象端末：PC専用（最低幅 1280px・スマホ／タブレット対応なし）

## 認証仕様（第4段階時点）

- ログイン方式：マスタPIN（4桁数字）
- 第4段階は **モック認証**：4桁数字なら何でも通る
- セッションは `localStorage` に `ultra-z-admin.session` キーで保存
- セッション有効期限：**8時間**（経過後は再ログイン要求）
- ログアウトボタンでセッション破棄→ログイン画面へ戻る

第5段階以降、`js/auth.js` の `login()` 関数内の「差し替え区間」を
`AdminApp.callMasterGas('login', { pin })` 呼び出しに置き換えることで
本認証へ移行する。

---

## マスタGAS接続

- `js/app.js` の `MASTER_GAS_URL` 定数に Web アプリURLを文字列で埋め込む
- 第4段階リリース時点では **空文字**（モック動作のみ）
- ラッパー関数 `AdminApp.callMasterGas(action, payload)` 経由で呼び出す
- マスタGASは `doPost({action, payload, ts})` の形式で受信する想定

## 用語ルール（厳守）

- 「店舗」「事業者」等の業態固定名称は使用しない
- 出退勤の表記は **出勤／退勤** に統一（「入店／退店」表記禁止）

## ターゲット社側 手動操作メモ

第4段階完了後に以下をターゲット社側で実施：

1. マスタGAS（第2段階で構築済み）を「ウェブアプリとしてデプロイ」
   - 実行ユーザー：自分（k@tgx.jp）
   - アクセスできるユーザー：全員（匿名も含む）
2. 取得した Web アプリURLを `js/app.js` の `MASTER_GAS_URL` に貼り付け
3. ローカルで再コミットして `git push origin main`
4. GitHub Pages 反映後、ログイン画面で4桁PINを入力して挙動確認
